学习freertos使用

“FreeRTOS 是一个开源、轻量级、确定性强的实时内核，它通过基于优先级的抢占式任务调度，在内存受限的嵌入式设备上实现高效的多任务并行管理。文档信息请参考 ——» freertos.org” 核心调度算法：抢占式优先级 FreeRTOS 采用基于优先级的抢占式调度逻辑，其运作遵循以下准则： 绝对优先：调度器确保处于就绪态的最高优先级任务始终获得 CPU 使用权。 即时抢占：一旦高优先级任务就绪，内核立即触发上下文切换，无条件暂停低优先级任务。 开发者义务：由于这种“强权”机制，开发者必须确保高优先级任务能及时调用 API（如 vTaskDelay 或信号量等待）进入阻塞态释放 CPU，否则将导致低优先级任务饥饿。 实时系统任务布局实例 根据任务的实时性（Deadline）与后果严重性分配优先级： 任务名称 优先级 职责描述 分配逻辑 vControlTask 最高 执行控制算法 截止时间最严苛，关系系统稳定，必须优先响应。 vKeyHandlerTask 中等 键盘输入处理 响应用户交互，实时性要求次于控制任务。 vLCDUpdateTask 最低 更新显示/后台 刷新频率低，允许在系统空闲时异步执行。 Idle Task 0 (最低) 系统维护/省电 内核自动创建，仅在无其他任务运行态时执行。 上下文切换原理 任务对调度器的存在是“无感知”的，其执行的连续性通过私有堆栈实现： 环境保存（换出）：任务被切出时，当前 CPU 的寄存器状态（PC、SP 等）会被压入该任务的私有堆栈中。 环境恢复（换入）：任务重新获得执行权时，调度器从其堆栈中弹出保存的现场信息，精确还原到被中断的一瞬。 独立性：每个任务通过专属堆栈维持自身的逻辑完整性。
